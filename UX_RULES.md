# Golf Swing Analyzer – UX Rules

## Core Principles

* Simplicity > features
* Show, don't explain
* One primary action per screen
* No clutter, no dashboards

## Hard Constraints

* Max 5 metrics visible at once
* No charts/graphs in MVP
* No more than 2 levels of navigation
* All feedback must be readable in <5 seconds

## Interaction Rules

* Tap targets must be large (thumb-friendly)
* Default flows must require zero setup
* Recording must start within 1 tap

## Feedback Rules

* Plain English only
* No technical jargon (e.g., avoid "internal rotation velocity")
* Always include "what to fix" not just "what's wrong"

## Visual Design

* Minimal UI (lots of whitespace)
* Neutral background (dark or muted)
* Highlight only what matters (angles, joints)
* Subtle motion only (no flashy animations)
